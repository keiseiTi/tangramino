# 介绍

**Tangramino** 是一个现代化的、模块化的、Schema 驱动的可视化编辑器框架。它的诞生旨在解决低代码平台开发中的通用问题：核心逻辑与视图的耦合、扩展性差、维护困难等。

## 为什么选择 Tangramino？

在构建可视化编辑器（如页面搭建、表单设计、流程编排）时，开发者往往面临着巨大的重复工作量：

- 需要设计一套 JSON 协议来描述页面。
- 需要实现复杂的拖拽交互（排序、嵌套、吸附）。
- 需要管理撤销/重做（Undo/Redo）的历史记录。
- 需要处理组件之间的联动逻辑。
- 需要保证编辑器的高性能渲染。

Tangramino 将这些通用能力封装在底层核心包中，让你专注于业务逻辑和物料开发。

## 核心设计理念

### 1. Schema 驱动 (Schema-Driven)

Tangramino 认为 **Schema 是应用的核心**。所有的页面结构、属性、样式、逻辑都应该被描述为一份标准的 JSON 数据。这种设计使得应用具有天然的跨平台能力，并且易于存储和传输。

### 2. 逻辑与视图分离 (Logic-View Separation)

- **Engine (`@tangramino/engine`)**：负责 Schema 的管理、事件分发、数据流转。它是纯 JS/TS 实现的，不依赖任何 UI 框架。
- **View (`@tangramino/react`)**：负责将 Engine 中的数据渲染为界面。目前我们首选支持 React，但未来可以轻松扩展到 Vue 或其他框架。

### 3. 模块化与可插拔 (Modular & Pluggable)

Tangramino 采用 Monorepo 架构，将功能拆分为多个独立的包：

- **基础层**：`engine` (核心逻辑), `react` (视图绑定)
- **组件层**：`base-editor` (通用拖拽编辑器), `flow-editor` (流程图编辑器)

你可以根据需求按需选用。例如，如果你只需要一个流程图工具，可以直接引入 `@tangramino/flow-editor`，而无需引入整个页面搭建的重型依赖。

## 适用场景

- **低代码/无代码平台**：快速搭建企业级页面搭建系统。
- **表单设计器**：构建复杂的动态表单配置工具。
- **流程编排工具**：工作流引擎、逻辑编排、数据流处理。
- **营销活动页搭建**：灵活的 H5 活动页制作工具。
